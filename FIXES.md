# Plotrix Full Audit

CRUCIAL
Security: isSafeExpressionInput uses a regex allowlist but the math.parse/compile pipeline still executes arbitrary JS-like expressions in the mathjs sandbox. The custom regression worker calls compiled.evaluate with user-controlled symbol names directly as object keys — prototype pollution is possible if a user names a variable "__proto__" or "constructor". Sanitize variable names against a strict allowlist before passing to evaluate().
Security: base64UrlDecode in graph.svelte.ts calls atob on untrusted URL hash content with no length cap. A crafted URL with a multi-megabyte hash will freeze the main thread. Add a byte-length guard before decoding.
Security: importJSON calls restoreSnapshot which calls createEquation for every equation in the payload. There is no rate-limit or size check on the number of equations beyond the 50-item slice in validateEquations, but regressionResults and dataSeries have no hard cap on row count beyond 2000 rows × 8 cols. A crafted JSON can allocate ~128 MB of strings. Add caps.
Performance: The canvas renderer calls getComputedStyle(document.documentElement) on every syncTokenCache() call, which is triggered inside every paint() frame. On fast machines this is a forced style recalc every rAF. Cache the result and only re-read when a MutationObserver detects a theme attribute change.
Performance: drawInequalityShading and drawScatterSeries create new OffscreenCanvas instances on every cache miss without pooling. Large canvases at 3× DPR create and GC huge buffers repeatedly. Pool or reuse off-screen buffers.
Performance: The analysis worker is re-instantiated inside ensureAnalysisWorker on every call until it exists, but the worker is never terminated on GraphState destruction. This leaks the worker if the component is torn down (e.g. HMR, SSR, test environments).
Performance: sampleEquation is called inside fitAll in a synchronous tight loop over all equations on the main thread. This can block for hundreds of milliseconds with complex equations at high zoom. Move fitAll sampling to the analysis worker.
Performance: In CanvasRenderer.paint(), drawInequalitySystems builds a per-inequality OffscreenCanvas for the intersection mask using a sequential createCacheLayer → drawImage → source-in pipeline. Each inequality adds a full-canvas compositing pass. Use a single-pass shader-style approach with clip regions instead.
Performance: The criticalPointCache and intersectionCache in graph.svelte.ts are plain Maps that grow unbounded. They are only cleared on viewport changes via clearAnalysisCaches(). Under rapid panning/zooming, thousands of stale entries accumulate. Apply an LRU eviction with a hard cap (e.g. 200 entries).
Performance: adaptiveSample calls refineCartesian recursively with depth up to 8 and budget up to maxN. For pathological functions (e.g. sin(1/x) near 0), this recursion exhausts the budget on a tiny visible segment, causing dropped frames. Add a per-segment point cap and abort refinement early.
UX: There is no loading/empty state on the graph canvas while equations are being parsed on mount. The canvas flashes blank for ~200ms before the first paint. Show a skeleton or keep the previous frame.
UX: The equation editor (ExpressionEditor) uses CodeMirror but has no math-aware autocomplete, no bracket matching highlight, and no syntax error underline. Users type blind with only a delayed error badge as feedback.
UX: Clicking "Analyze" on an equation shows a shimmer indefinitely if the analysis worker returns null (e.g. on a non-cartesian equation). The loading state is never resolved to an error message — the panel just spins forever.
UX: On mobile (< 960px), the sidebar is a fixed overlay but there is no swipe-to-open/close gesture. The only affordance is the topbar toggle button, which is easy to miss. Add a swipe gesture on the canvas edge.
UX: The "Shade between" button only appears when exactly 2 equations are selected, but there is no affordance or tooltip explaining how to multi-select (Shift+click). New users will never discover this feature.
HIGH
Styling: The topbar uses `overflow-x: auto` on mobile, which causes the toolbar icons to scroll horizontally and disappear off-screen without any scroll indicator. Group rarely-used actions into an overflow menu instead.
Styling: The sidebar panel uses `backdrop-filter: blur(12px)` but its parent `.sidebar` has `overflow: hidden`, which clips the blur to the sidebar boundary and causes a hard edge artifact against the canvas. Remove the overflow:hidden or wrap the blur target differently.
Styling: The `.workspace` grid transition animates `grid-template-columns` which is not GPU-composited and causes layout thrash on every frame of the sidebar open/close animation. Use `transform: translateX` on the sidebar instead and keep the grid static.
Styling: `.card` (EquationCard) uses CSS `transition` on `border-color`, `box-shadow`, and `transform` simultaneously on hover. Combined with the KaTeX render inside, this causes composite layer promotion of every equation card. Only promote the active/hovered card.
Styling: The analysis drawer (AnalysisPanel) hardcodes `width: 320px` with no minimum width and no responsive adjustment. On screens narrower than 700px it overlaps the entire canvas and has no close swipe.
Styling: `--space-1-5` is used in `+page.svelte` (`.sidebar-resizer`) but is never defined in tokens.css. This silently resolves to 0. Define it as 6px or replace with a defined token.
Styling: The `.settings-list` uses `grid-template-columns: repeat(2, minmax(0, 1fr))` which causes two narrow columns on mid-range viewports (960–1200px) where the modal is 70% wide. The columns become too narrow to read comfortably. Switch to a single column below 1100px.
Styling: The export modal grid `repeat(2, minmax(0, 1fr))` stacks the SVG and JSON cards alone on the bottom row (5 items, 2-col grid), creating an asymmetric orphan layout. Use `repeat(auto-fill, minmax(140px, 1fr))` or 3 columns.
Styling: `EquationCard` `.summary-row` uses a 6-column fixed grid `24px 10px minmax(0,1fr) 24px 24px 28px`. On narrow sidebars (300px), the preview text column gets crushed below ~150px. The equation preview truncates with no ellipsis. Add `text-overflow: ellipsis; overflow: hidden` to `.preview`.
Styling: The color swatch in the sidebar is 10px wide and 42px tall — a non-standard affordance. The ColorPicker popover opens on top of it but the trigger area is tiny and hard to hit on touch. Widen the swatch to at least 16px or make the whole row clickable.
Visual Hierarchy: The topbar has no product wordmark/logo visible by default — the brand lockup is inside the sidebar. When the sidebar is collapsed, the app has no identity in the topbar. Move a minimal logo to the topbar left.
Visual Hierarchy: The equation cards have three competing action zones: the visibility toggle, the color swatch, and the expand chevron. The duplicate/delete buttons are hidden in the footer below the fold. Users routinely look for delete and can't find it. Promote delete to the summary row (icon only, destructive color on hover).
Visual Hierarchy: The "Add" and "Import" buttons in the sidebar header are the same visual weight. "Add" should be primary (filled) and "Import" should be tertiary (ghost text only).
Visual Hierarchy: The VariableSliderPanel appears below the equation list with no clear visual separation from the equations. It looks like another equation card. Use a distinct section background or a labeled divider.
Visual Hierarchy: The analysis panel sections (Domain, Zeros, etc.) use accordion buttons with no open/closed indicator icon position that is standard. The chevron is missing entirely from the accordion buttons in AnalysisPanel — there is only a text label. Add chevron icons.
UX: Pressing Tab in the canvas to cycle equations is a non-standard pattern and conflicts with normal focus management. Users tabbing through the page expect focus to move to interactive elements, not cycle graph equations.
UX: The command palette has no category grouping. All 12 actions appear in a flat list with no section headers (View, Analysis, Export, etc.), making scanning slower.
UX: The RegressionPanel "Fit" button is disabled while `!dataset` but shows no tooltip or reason why. If no data is plotted, the user has no idea why the button is greyed out.
UX: The DataPanel has no row deletion affordance. Users can only clear cells individually. There is no way to delete a row or reset the sheet without re-importing.
UX: Closing uses the symbol "X" instead of an icon, change closing globally to use icon instead of a hard-coded symbol.
UX: There is no keyboard shortcut or button to close the AnalysisPanel drawer other than clicking the X button or pressing Escape. The backdrop click closes it but there is no hint of this.
UX: The topbar toolbar buttons have tooltips with 200ms delay (IconButton), but on mobile/touch there is no tooltip at all and the icons have no visible labels. Touch users cannot identify buttons by icon alone.
UX: The "Fit all" button on the canvas zoom pod is labeled "Fit" with no icon, while the topbar has the same action as an icon with no label. These are inconsistently presented.
UX: Equation reorder up/down arrow buttons are inside the card's summary row, but drag-and-drop reorder is also available via the entire summary row. The two mechanisms conflict — drag starts on mouse down, but the up/down buttons are inside the draggable zone. Dragging near the up button frequently triggers drag instead of click.
MEDIUM
Performance: KaTeX rendering is done synchronously in `$derived` blocks (equationHtml, previewHtml). For pages with 10+ equations, this blocks the reactive update loop. Use a web worker, implement it fully and properly!
Performance: The `sharedKatexCache` in EquationCard and AnalysisPanel is a module-level Map with no eviction. It grows unbounded across the session. Cap it at ~200 entries with LRU.
Performance: `graph.variableScope()` memoizes by joining variable name:value pairs into a string key. With many variables, this creates a new string on every call even when nothing changed. Use a shallow equality check on the variables array instead.
Performance: In `drawPolarCurve`, maxSteps is recalculated on every render as `Math.min(4000, Math.round(scaleX * 120))`. At high zoom, this can be 4000 steps × trig per step = slow. Cache the computed curve in the same manner as scatter (keyed by equation raw + viewport bucket).
Performance: `detectVerticalAsymptotes` in equationAnalysis.ts samples 400 points from -100 to 100 with a nested inner loop of 21 probes each = up to 8400 evaluations per analysis run. This is excessive. Use a smarter sign-change + derivative spike approach.
Performance: The regression worker sends Float64Array with transferable buffers but the result RegressionResult is posted back as a plain object (JSON-serialized), which copies all coefficient arrays. Use transferable ArrayBuffers for the result coefficients too.
Performance: `drawCurve` recalculates `baseSamples` and `maxSamples` from `frameBudget` on every render frame. These are deterministic given `visibleCount` and `surface.width`. Memoize them.
Styling: The `--space-1-5` token (used in sidebar resizer margin) is undefined. All other spacing tokens jump from `--space-1` (4px) to `--space-2` (8px). Add `--space-1-5: 6px` to tokens.css.
Styling: `color-mix(in srgb, ...)` is used throughout but has no fallback for browsers that do not support it (Safari < 16.2, Chrome < 111). Add `@supports` fallbacks or use pre-computed values.
Styling: The modal `.panel` uses `width: 70%` with no `max-width`. On 4K displays this creates an uncomfortably wide modal. Add `max-width: 860px`.
Styling: The `.drawer` (AnalysisPanel) has `transition: transform 280ms ease-in-out` but the backdrop fade is instant (no transition). The backdrop and drawer animate at different speeds, creating a jarring mismatch.
Styling: The toast viewport is `position: fixed; bottom: var(--space-6); right: var(--space-6)` with `width: min(360px, calc(100vw - 2rem))`. On mobile with the sidebar overlay open, toasts are hidden behind the sidebar. Use z-index above the sidebar.
Styling: The `.zoom-pod` buttons have `border-radius: 16px` hardcoded. This should use a token (`--radius-2xl` or `--radius-full`) to stay consistent with the design system.
Styling: The shimmer animation in AnalysisPanel uses `background-size: 200% 100%` and animates `background-position`. This is GPU-friendly but the four skeleton divs all animate in sync (same keyframe timing). Stagger them with `animation-delay` to look more natural.
Styling: `.settings-category-header` uses a `<button>` but has no `:focus-visible` style override — it falls through to the global `:focus-visible` which applies `box-shadow: var(--shadow-focus)` correctly, but the button background is already styled with a gradient, so the focus ring is barely visible against the light theme header. Increase contrast.
Inconsistencies: The ExpressionEditor uses `font-size: 1.15rem` (hardcoded rem) while everywhere else uses CSS custom property tokens (`--text-sm`, etc.). Normalize to tokens.
Inconsistencies: The equation card footer buttons (Duplicate, Delete) use `font-size: var(--text-xs)` while the expand/collapse details section uses `var(--text-sm)`. Card internal font sizes are inconsistent.
Inconsistencies: Icon sizes are inconsistent: some IconButton SVGs are 18px, some in EquationCard are 14px, 16px, 18px, and 24px all within the same component. Define a small set of icon size tokens.
Inconsistencies: `formatSig` is defined independently in criticalPoints.ts, equationAnalysis.ts, intersections.ts, and canvas.ts — four separate identical implementations. Extract to utils/format.ts.
Inconsistencies: The color palette `COLOR_PALETTE` is duplicated verbatim in graph.svelte.ts and ColorPicker.svelte's PRESET_COLORS (partially overlapping). Single source of truth.
Inconsistencies: `clamp` is imported from `$utils/format` in some files and reimplemented inline in canvas.ts as a local import from the same path. The canvas.ts also imports it correctly — this is fine, but several other files call `Math.min(Math.max(...))` inline instead of using the shared clamp. Normalize.
Bad Code: In EquationCard, the "Plot f'(x)" button constructs a numerical derivative string by string-replacing 'x' with '(x+0.0001)' in the raw equation string. This breaks for parametric/polar equations and for multi-character expressions containing 'x' as a substring (e.g. "exp(x)" becomes "exp((x+0.0001))"). Use the engine's derivative function or a safer substitution (choose what's best for implementing for UX, not for your speed).
Bad Code: `detectPeriod` in equationAnalysis.ts tries 20 candidate periods by dividing the visible span by integers 1–20. This never detects periods shorter than span/20 or longer than span. The result is always null for trig functions at unusual zoom levels.
Bad Code: `symbolicIntegral` in equationAnalysis.ts is a hardcoded if/else chain matching regex patterns on the node toString output. It covers only ~7 patterns and returns "numerical (Simpson's rule)" for everything else without actually computing a Simpson's rule result. Implement fully complete numerical integration.
Bad Code: The `sinusoidalRegression` gradient descent uses a hardcoded learning rate of `1e-4 / Math.max(1, x.length)` with momentum 0.9 and runs for exactly 500 iterations with no early stopping. This is slow for large datasets and may not converge for non-sinusoidal data. Use a proper Levenberg-Marquardt step (already implemented for custom regression) instead.
Bad Code: `inferDomain` samples 2000 points from -1000 to 1000 regardless of the current viewport. For slow functions this is expensive and the result is viewport-independent anyway. Sample a wider range with fewer points (e.g. 200 points over ±1e6) since domain inference is coarse by nature.
Bad Code: In interactions.ts, both `Mod+Y` and `Mod+Shift+Z` are mapped to redoHistory but the first branch also catches `Mod+Z+Shift` before the second branch, making the second branch unreachable. The condition `(modifier && lower === 'y') || (modifier && lower === 'z' && event.shiftKey)` in the second if block is never hit because the `Mod+Shift+Z` redo is already handled by the first if block. Remove the duplicate.
Bad Code: `graph.setEquationRenderTime` has an early return when `!graph.settings.showRenderTime`, meaning render times are never collected unless the setting is on. But `formatDuration` is still called in EquationCard for all equations - it just always shows "0 ms". Conditionally render the element.
LOW
UX: There is no "empty equation" placeholder text in the graph canvas when no equations exist. The canvas just shows a blank grid with no CTA.
UX: The "Copy report" button in AnalysisPanel copies a plaintext report but provides no visual confirmation (toast, checkmark, etc.) that the copy succeeded.
UX: The share URL input in the Share modal is `readonly` but not styled differently from editable inputs (no background color change, no lock icon). Users may try to edit it.
UX: The zoom pod "Fit" and "Reset" text buttons have no icons. All other zoom pod buttons are icon-only. The text buttons are wider and break the pod's visual rhythm. Add icons and optional labels.
UX: Keyboard shortcut "0" to reset view triggers even when the user is typing "0" in a variable min/max input field (the isFormField guard in handleKeyDown catches HTMLInputElement, so this should be safe — but contenteditable cells in DataPanel are not caught). Add `target?.closest('[contenteditable]')` to the guard.
UX: The "Shade between" feature adds two inequality equations with hardcoded `y >=` / `y <=` prefixes. If the user's selected equations have spaces or special characters in the raw string, the generated inequality may be syntactically invalid.
Styling: The `.equation-list` uses `padding-right: var(--space-1)` to offset the scrollbar, but on macOS with overlay scrollbars this creates unwanted right padding. Use `scrollbar-gutter: stable` instead.
Styling: The data panel `.sheet` table uses `table-layout: fixed` but the `Add column` th has no explicit width, causing it to collapse to nearly 0px. Give it a fixed `width: 40px`.
Styling: The `VariableSliderPanel` config row uses `grid-template-columns: 1fr auto 1fr auto` which allocates equal space for min and max inputs but the "to" and "step:" text labels shrink unpredictably on narrow sidebars. Use explicit widths for the text labels.
Styling: `CommandPalette` backdrop uses `backdrop-filter: blur(12px)` which on Firefox (without hardware acceleration) causes the entire page to repaint on every frame while the palette is open. Provide a fallback solid background with `@supports not (backdrop-filter: blur())`.
Styling: The `.analysis-panel .drawer` uses `overflow: auto` with no `-webkit-overflow-scrolling: touch`, causing momentum scrolling to be absent on iOS Safari.
Styling: The progress/shimmer animation in AnalysisPanel uses `linear` timing which gives a mechanical look. Use `ease-in-out` for a smoother shimmer.
Performance: `viewportBucket()` in canvas.ts is called multiple times per paint frame (once per shading cache key, once per scatter cache key, once per system cache). Each call does 4 Math.round operations. Compute once per paint and pass down.
Performance: The `inequalitySystemCache`, `shadingCache`, and `scatterCache` in CanvasRenderer each cap at 20 entries. But 20 OffscreenCanvas objects at 1920×1080×4 bytes each = ~160 MB just for caches. Instead of capping by count (20), cap by total memory usage (e.g., 64MB or 128MB total). Mechanism: Keep an LRU (Least Recently Used) cache. Calculation: Calculate the size of an OffscreenCanvas upon creation: width * height * 4. Cleanup: When adding a new entry, if total_cache_size > target_memory_limit, remove the oldest canvases until you are under the limit. This allows caching many small elements but fewer large ones.
Performance: `toSVGString()` returns a minimal SVG with no actual curve data — it's just a background rect and a watermark text. This is misleading. Either implement proper SVG export using the sampled path data, DO NOT rename the button to "Export (placeholder)".
Bad Code: `base64UrlEncode` manually converts a string to UTF-8 bytes using TextEncoder, then builds a binary string character-by-character in a for...of loop before calling btoa. For large session exports this is slow. Use `btoa(String.fromCharCode(...new Uint8Array(bytes)))` with a chunked approach.
Bad Code: `deserializeSnapshot` uses `source.trim().startsWith('{')` to detect JSON vs base64, but a base64-encoded string could theoretically start with `{` after decoding. The detection is fragile. Use a try/catch with JSON.parse first, fallback to base64.
Bad Code: The `DataPanel` CSV parser is a hand-rolled state machine that handles quoted fields but does not handle escaped quotes inside quoted fields (only `""` doubling, not `\"`). It also silently drops fields beyond MAX_CSV_COLUMNS without warning the user.
